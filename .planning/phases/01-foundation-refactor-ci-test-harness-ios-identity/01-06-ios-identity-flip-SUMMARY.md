## Decisions Made

- **Deployment target 14.2 vs 14.0:** Locked at 14.2 per CONTEXT.md D-15. One notch above 14.0 captures App Attest stability fixes from the 14.0 launch regression window.
- **--platforms ios only in flutterfire configure:** MentorMinds v1.0 is iOS-only; macOS config in the old firebase_options.dart was vestigial. Dropped intentionally.
- **APNs .p8 deferred:** Phase 1 does not use push notifications (firebase_messaging SDK is installed but not imported). Phase 6 is the correct place to re-associate APNs .p8 with the new iOS app registration. Documented in BACKEND_SETUP.md step 3.
- **Orphan Firebase iOS apps:** 3 registrations remain in the Firebase Console (`com.MentorMind`, `com.arnobrizwan.mentorminds`, `com.mentormind`) — Firebase CLI does not support app deletion; must be removed via Firebase Console manually before GA. Not a blocker for v1.0 development.

- **Files modified:** ios/Runner/GoogleService-Info.plist (correct version placed)

**2. 3 orphan iOS app registrations in Firebase Console**
- **Found during:** Post-checkpoint analysis
- **Issue:** Three stale iOS app registrations exist in Firebase project `mentor-mind-aa765`: `com.MentorMind` (original bad bundle ID), `com.arnobrizwan.mentorminds` (second attempt), and `com.mentormind` (typo attempt). Firebase CLI does not support app deletion.
- **Resolution:** Not a blocker for v1.0. Marked for manual cleanup via Firebase Console before GA. Documented here and in BACKEND_SETUP.md.
- **Files modified:** None (Console-only issue)

**3. flutter build ios --no-codesign fails — missing iOS 26.5 SDK platform**
- **Found during:** Task 2 Step D (smoke build)
- **Issue:** `flutter build ios --no-codesign --debug` fails with "Unable to find a destination matching the provided destination specifier: { generic:1, platform:iOS }" because iOS 26.5 platform SDK is not installed in Xcode 26.5. Both `--no-codesign` and `--simulator` flags exhibit the same failure.
- **What succeeded:** Pod install ran successfully in 19.9s during the build attempt. "Xcode build done" appeared (the compilation phase completed). IPHONEOS_DEPLOYMENT_TARGET = 14.2 confirmed in Pods build settings. The failure is at destination resolution, not compilation.
- **Resolution:** Environment-level issue — Xcode 26.5 requires separate iOS 26.5 SDK download via Xcode > Settings > Components. Not related to bundle ID or deployment target changes. Documented as known environment constraint. To validate the build: open `ios/Runner.xcworkspace` in Xcode 26.5 and install the iOS 26.5 SDK, or connect a device running iOS 26.5.
- **Per CONTEXT.md D-15:** "if any pod requires bumping, do it inside PR-3" — all Firebase ^5.x, google_sign_in ^6.x, connectivity_plus ^6.x, image_picker ^1.x confirmed compatible with 14.2 per RESEARCH Assumption A2. No pod required bumping.
